## Why

編輯訂單時，總金額欄位一旦被手動調整，就不會再跟著既有的金額計算邏輯自動回填。現在若要恢復成系統原本的計算結果，只能人工重算並手動輸入，容易出錯，也會拖慢訂單修正流程。

## What Changes

- 在編輯訂單 modal 的總金額欄位旁新增一個「恢復原計算」按鈕。
- 按鈕會使用目前已選的合作廠商、車輛、開始時間、結束時間，套用既有的前端金額計算邏輯，將總金額欄位覆寫為計算結果。
- 此操作只影響目前表單中的 `payment_amount` 值，不新增 API，也不變更後端儲存格式。
- 若資料不足而無法計算有效金額，系統不得覆蓋既有手動輸入值，並需提供明確提示。

## Capabilities

### New Capabilities
- None.

### Modified Capabilities
- `order-amount-recalculation`: 編輯訂單流程需提供一個明確的 UI 操作，讓使用者可用既有計算邏輯恢復總金額欄位。

## Impact

- **system/backend/components/AddOrderModal.tsx**: 調整總金額欄位 UI、加入恢復按鈕與按鈕事件處理。
- 既有前端金額計算函式需可被按鈕重用，且在恢復成功後同步更新手動編輯狀態。
- 無資料庫異動，無 API 合約變更。
